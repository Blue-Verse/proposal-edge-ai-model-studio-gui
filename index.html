프로젝트 소개

파편화된 AI 모델 학습·변환·평가 작업을 하나의 직관적인 Web GUI로 통합합니다. FastAPI 기반 API Gateway가 기존 sv-network Docker 컨테이너(YOLO·SNPE·CDFSOD)를 REST·WebSocket으로 제어하여, 비전문 작업자도 데이터셋 구성부터 Few-Shot 학습, NPU 보드 평가까지 한 화면에서 수행하는 통합 대시보드를 구축합니다.

핵심 요구사항 분석

  • FastAPI API Gateway — 기존 sv-network Docker 컨테이너(YOLO·SNPE·CDFSOD)를 REST·WebSocket으로 제어하는 백엔드 게이트웨이 구축
  • 데이터 분석·관리 — YAML 기반 데이터셋 통계(이미지 수·클래스 분포) 분석·차트 시각화, 서버 내 파일 브라우저 UI
  • 학습·추론 제어 — YOLO(Standard·KD)·Few-Shot(CDFSOD) 파라미터 설정, Job Queue 관리, WebSocket 실시간 학습 로그 터미널, 단건·배치 추론 및 결과 이미지 렌더링
  • 변환·SNPE 양자화 제어 — ONNX/DLC 변환·INT8 양자화 파라미터 설정, Qualcomm NPU 보드 원격 연결(SSH/SCP) 및 mAP 평가 연동
  • Edge 통합 데이터 파이프라인 — 데이터 정제(품질·유사도 검사) 파라미터 설정·실행, CVAT 라벨링 툴 연동 인터페이스
  • 산출물 — Frontend·Backend 소스 원본, API 명세서·시스템 구성도, Dockerfile·배포 가이드

제안 포인트

  • 과업 범위 정확 이해 — 본 건은 AI 모델 자체 개발이 아닌, 이미 준비된 AI 엔진을 제어하는 '웹 대시보드 + API Gateway' 구축임을 명확히 인지하고 그에 최적화된 제어 레이어 아키텍처를 제안합니다.
  • FastAPI 비동기 아키텍처 — Uvicorn 기반 비동기 게이트웨이로 장시간 학습 Job을 논블로킹 처리하고, WebSocket으로 실시간 로그·진행률을 끊김 없이 스트리밍합니다.
  • 기존 Docker 네트워크 무중단 연동 — sv-network 내 컨테이너 통신 규약을 준수하여 기존 인프라 변경 없이 REST·WebSocket 제어 레이어만 추가합니다.
  • 비전문가 친화 UX — Bootstrap 5 기반 직관적 대시보드로 데이터셋 구성·학습·양자화·평가 전 과정을 GUI화하여 CLI 의존도를 제거합니다.
  • 유사 프로젝트 경험 기반 효율 개발 — 실시간 WebSocket 대시보드, AI 엔진 오케스트레이션, 대규모 API Gateway를 다수 수행한 경험으로 산출물 완성도와 일정 준수를 동시에 보장합니다.

PRD (Product Requirements Document)

프로젝트 요구사항을 정의하고 기능 범위를 명확히 합니다.

1. 프로젝트 배경 및 목적

현재 AI 모델 학습 및 변환 작업이 개별 스크립트·CLI 환경에서 파편화되어 진행되고 있습니다. 본 프로젝트는 이를 하나의 직관적인 Web GUI 플랫폼(SV Model Studio)으로 통합하여, 전문 지식이 부족한 작업자도 데이터셋 구성, Few-Shot 학습, NPU 보드 평가를 손쉽게 수행하도록 업무 효율성을 높이는 것을 목표로 합니다. 본 과업은 AI 모델 자체 개발이 아니라, 기존 AI 엔진(YOLO·SNPE·CDFSOD) Docker 컨테이너를 제어하는 웹 대시보드 및 FastAPI API Gateway 구축에 한정됩니다.

3. 기능 요구사항

3.1 데이터 분석·관리

  • [필수] YAML 기반 데이터셋 통계(이미지 수·클래스 분포) 분석 및 차트 시각화
  • [필수] 서버 내 파일 브라우저(경로 탐색·선택) UI

3.2 AI 모델 학습·추론 제어

  • [필수] YOLO(Standard·KD)·Few-Shot(CDFSOD) 파라미터 설정 UI
  • [필수] FastAPI Job Queue 관리 + WebSocket 실시간 학습 로그 터미널 출력
  • [필수] 단건/배치 추론 실행 및 결과 이미지 렌더링

3.3 모델 변환·SNPE 양자화 제어

  • [필수] ONNX/DLC 변환·INT8 양자화 파라미터 설정 UI
  • [필수] Qualcomm NPU 보드 원격 연결(SSH/SCP) 및 mAP 평가 실행 연동

3.4 Edge 통합 데이터 파이프라인 제어

  • [필수] 데이터 정제(품질·유사도 검사) 파라미터 설정·실행 연동
  • [필수] CVAT 라벨링 툴 연동 인터페이스

5. 기술 제약사항

  • 백엔드는 반드시 FastAPI(Python·Uvicorn) 기반으로 구축
  • 기존 sv-network Docker 네트워크 및 AI 컨테이너 API 규약 준수
  • 프론트엔드는 Bootstrap 5 기반 SPA (Vanilla JS / React / Vue 중 협의 선택)
  • 제공되는 기술명세서(PDF)·AI 컨테이너 API 가이드 기준 구현

6. 범위 외 사항 (Out of Scope)

  • AI 모델 자체의 개발·학습 알고리즘 구현 (기존 엔진 제어만 수행)
  • AI 엔진(YOLO·SNPE·CDFSOD) 컨테이너 내부 로직 개발
  • NPU 하드웨어·보드 펌웨어 개발
  • CVAT 등 외부 툴 자체 기능 개발 (연동 인터페이스만 구현)
